# REQUIREMENTS

## 1. 문서 목적

이 문서는 TimeTracker Backend의 1차 MVP 범위를 정의합니다.  
개발 중 기능 추가나 과도한 설계를 방지하고, 현재 반드시 구현해야 할 범위를 고정하는 것을 목적으로 합니다.

## 2. 시스템 개요

사용자의 시간 기록 데이터를 저장하고,  
이를 기반으로 기본적인 시간 사용 통계를 제공하는 백엔드 시스템입니다.

1차에서는 데이터 수집, 저장, 조회, 기본 통계 제공까지를 목표로 하며,  
AI 분석 및 추천 기능은 포함하지 않습니다.

## 3. 사용자 유형

### 일반 사용자
- 회원가입
- 로그인
- 시간 기록
- 기록 조회
- 통계 확인
- 카테고리 관리

※ 관리자 기능은 1차 범위에서 제외합니다.

## 4. 1차 기능 요구사항

### 4.1 인증 기능
- 회원가입
- 로그인
- 사용자 인증 처리
- 인증 기반 API 접근 제어

### 4.2 시간 기록 기능
- 카테고리 선택 후 기록 시작
- 기록 종료
- 실행 중 상태 조회

#### 비즈니스 규칙
- 한 번에 하나의 타이머만 실행 가능
- 실행 중 타이머가 있으면 시작 불가
- 종료 시 실행 중 타이머가 없으면 오류
- 종료 시 시작 시간보다 종료 시간이 빨라서는 안 됨

### 4.3 카테고리 관리
- 카테고리 생성
- 카테고리 조회
- 카테고리 수정
- 카테고리 삭제

### 4.4 기록 조회
- 날짜 기준 기록 조회

### 4.5 데이터 저장
저장 대상 데이터는 아래와 같습니다.

- 사용자 ID
- 시작 시간
- 종료 시간
- 카테고리
- 상태값

### 4.6 통계 기능
- 일별 총 시간 조회
- 주간 총 시간 조회
- 카테고리별 시간 집계

## 5. 제외 요구사항

아래 기능은 1차에서 구현하지 않습니다.

- 집중도 입력
- 낭비 시간 분석
- 추천 시스템
- LLM 리포트 생성
- 군집 분석
- 자동 기록
- 관리자 기능

## 6. 비기능 요구사항

### 6.1 성능
- 기록 요청 응답 시간 1초 이내를 목표로 합니다.

### 6.2 신뢰성
- 데이터 손실을 방지해야 합니다.
- 잘못된 시간 입력을 방지해야 합니다.
- 실행 중 상태와 종료 상태가 일관되게 유지되어야 합니다.

### 6.3 보안
- 사용자별 데이터 접근 제한
- 인증 기반 API 접근
- 본인 데이터만 조회 가능해야 함

### 6.4 사용성
- API 구조는 명확해야 합니다.
- 최소한의 입력만으로 기록이 가능해야 합니다.

## 7. 데이터 요구사항

### 7.1 수집 데이터
- 시작 시간
- 종료 시간
- 카테고리
- 사용자 ID

### 7.2 가공 데이터
- 일별 총 시간
- 주간 총 시간
- 카테고리별 합계

## 8. 시스템 구조

1차 시스템 구조는 아래와 같습니다.

- Backend: Spring Boot API
- DB: PostgreSQL
- ORM: JPA (Hibernate)
- 인증: JWT

## 9. 설계 원칙

- 패키지 구조를 유지합니다.
- DTO와 Entity를 분리합니다.
- FK 관계는 JPA 연관관계로 매핑합니다.
- status는 enum을 사용합니다.
- 과도한 추상화보다 동작하는 MVP 완성을 우선합니다.

## 10. 완료 기준

아래 조건을 만족하면 1차 MVP 완료로 봅니다.

- 회원가입 / 로그인 API가 동작한다.
- 카테고리 CRUD가 동작한다.
- 타이머 시작 / 종료 / 현재 상태 조회가 동작한다.
- 날짜 기준 기록 조회가 가능하다.
- 일별 / 주간 / 카테고리별 통계 조회가 가능하다.
- 예외 처리 결과가 일관되게 반환된다.
- 사용자 데이터가 인증 기준으로 분리된다.
- DB에 정상 저장 및 조회가 가능하다.

## 11. 우선순위

### 최우선
- 인증
- 시간 기록
- 카테고리 관리
- 통계 조회

### 그다음
- 예외 처리 정리
- Swagger 문서화
- 테스트 코드 보강

## 12. 범위 관리 원칙

새로운 기능을 추가하기 전 아래를 먼저 확인합니다.

- 지금 기능이 1차 MVP 범위에 포함되는가
- 현재 요구사항 완료에 직접 필요한가
- 지금 당장 구현해야 하는가, 아니면 추후 확장으로 미뤄도 되는가
